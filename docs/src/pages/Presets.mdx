## CrossFlicking

CrossFlicking supports flicking that can move in both horizontal and vertical directions.

You can easily create UIs with panels with multiple categories, like the example below.

<CrossFlickingDemo />

:::info
Moving the first panel up in the vertical Flicking changes to the previous category and changes the index in the horizontal Flicking.

Moving the last panel down in the vertical Flicking changes to the next category and changes the index in the horizontal Flicking.
:::

### Listening to Events

You can listen to events from the Flicking inside each Panel.
Each event has a `mainIndex: number` property, which is the index of the Panel including Flicking.
- sideHoldStart extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-holdStart"}>holdStart</Link>
- sideHoldEnd extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-holdEnd"}>holdEnd</Link>
- sideMoveStart extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-moveStart"}>moveStart</Link>
- sideMove extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-move"}>move</Link>
- sideMoveEnd extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-moveEnd"}>moveEnd</Link>
- sideWillChange extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-willChange"}>willChange</Link>
- sideChanged extends <Link to={"https://naver.github.io/egjs-flicking/ko/docs/api/Flicking#event-changed"}>changed</Link>

### Options

```ts
class CrossFlicking extends Flicking
```
|PARAMETER|TYPE|OPTIONAL|DEFAULT|DESCRIPTION|
|:---:|:---:|:---:|:---:|:---:|
|sideOptions|FlickingOption|✔️|{}|Options for Vertical Flicking instances.|
|preserveIndex|boolean|✔️|true|Preserves the previous vertical index when the horizontal index changes.|
|disableSlideOnHold|boolean|✔️|true|Prevents horizontal and vertical Flicking from being dragged at the same time.|
|disableIndexSync|boolean|✔️|false|Prevents index changes in the horizontal flicking from changing the index in the vertical flicking.|
